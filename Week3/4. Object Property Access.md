 4. Object Property Access
👉 This is the bridge between basic coding → real-world APIs & JSON

Objects are used to store real-world structured data:
let user = {
    name: "Rahul",
    age: 25,
    city: "Kolkata"
};

👉 This is exactly how APIs and JSON data look.

🧠 Big Idea
Concept
Meaning
Object
Collection of key-value pairs
Key
Property name
Value
Actual data


1️⃣ Dot Notation
👉 Concept
Access property using dot (.)

Example
let user = {
    name: "Rahul",
    age: 25
};

console.log(user.name);
console.log(user.age);

Output:
Rahul
25


🧠 When to Use
✔ When key is fixed and known
✔ Most common method

⚠️ Limitation
❌ Cannot use dynamic keys
❌ Cannot use spaces in key

2️⃣ Bracket Notation
👉 Concept
Access property using []

Example
console.log(user["name"]);
console.log(user["age"]);


🧠 Special Advantage
You can use:
Variables
Dynamic keys
Keys with spaces

Example with space
let user = {
    "full name": "Rahul Kumar"
};

console.log(user["full name"]);


3️⃣ Dynamic Keys (Very Important 🔥)
👉 This is where real developer thinking starts

Example
let user = {
    name: "Rahul",
    age: 25
};

let key = "name";

console.log(user[key]);   // dynamic access

Output:
Rahul


❌ Wrong Way
console.log(user.key);   // ❌ undefined

👉 Because it looks for property literally named "key"

🧠 Real-World Use
API response handling
Form inputs
Dynamic filtering
User settings

🔥 Real API Example
let response = {
    status: 200,
    data: {
        user: {
            name: "Amit",
            email: "amit@gmail.com"
        }
    }
};

Access:
response.data.user.name

👉 This is 100% real backend/frontend work

🔥 CRUD with Objects
Create
user.country = "India";

Read
console.log(user.name);

Update
user.age = 30;

Delete
delete user.city;


⚠️ Common Mistakes
❌ Using dot for dynamic key
user.key   // ❌ wrong


❌ Forgetting quotes in bracket
user[name] // ❌ error (name not defined)


❌ Accessing undefined property
user.salary // undefined
