6. Nested Data Structures
👉 This is where you start thinking like a real backend/frontend developer

🧠 Big Idea
👉 Real-world data is NOT flat
It is nested (inside inside inside)
User → Orders → Products → Details


1️⃣ Array Inside Object
👉 Example
let user = {
  name: "Rahul",
  orders: [101, 102, 103]
};


👉 Access
console.log(user.orders[0]); // 101


🧠 Real Use
User → list of orders
Student → list of marks
Cart → list of items

2️⃣ Object Inside Array
👉 Example
let users = [
  {name: "Rahul", age: 25},
  {name: "Amit", age: 30}
];


👉 Access
console.log(users[1].name); // Amit


🧠 Real Use
API user list
Product catalog
Employee records

3️⃣ Deeply Nested Structure (🔥 MOST IMPORTANT)
👉 Example
let data = {
  user: {
    name: "Rahul",
    orders: [
      {
        id: 1,
        items: [
          {product: "Laptop", price: 50000},
          {product: "Mouse", price: 500}
        ]
      }
    ]
  }
};


🔥 Deep Access Logic
👉 Get "Laptop"
data.user.orders[0].items[0].product


👉 Get price of Mouse
data.user.orders[0].items[1].price


🧠 Trick to Understand (VERY IMPORTANT)
Break step by step:
data
→ user
→ orders[0]
→ items[0]
→ product

👉 Always go one level at a time

🔥 Real API Example
let response = {
  status: 200,
  data: [
    {
      id: 1,
      profile: {
        name: "Amit",
        skills: ["JS", "React"]
      }
    }
  ]
};


👉 Access "React"
response.data[0].profile.skills[1]


⚠️ Common Mistakes
❌ Skipping levels
data.orders[0] ❌


❌ Wrong index
items[2] ❌ (doesn't exist)


❌ Undefined error
data.user.address.city ❌


💡 Pro Tip (Advanced)
Use safe checking:
if(data.user && data.user.orders){
   console.log(data.user.orders[0]);
}
