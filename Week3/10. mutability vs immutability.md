10. Mutability vs Immutability (Beginner Intro)
👉 This topic explains MANY beginner bugs and why your data behaves strangely

🧠 Big Idea
👉 There are 2 ways data behaves:
Mutable     → Can change original data
Immutable   → Cannot change, creates new copy


1️⃣ Mutable (Data CHANGES)
👉 Concept
Original data gets modified

Example (Array)
let arr = [10, 20, 30];

arr.push(40);

console.log(arr);

Output:
[10, 20, 30, 40]

👉 Original array changed ❗

Example (Object)
let user = {name: "Rahul"};

user.name = "Amit";

console.log(user);

👉 Same object changed

🧠 Real Meaning
Same memory → modified


2️⃣ Immutable (Data DOES NOT CHANGE)
👉 Concept
Original data stays same → new copy created

Example
let arr = [10, 20, 30];

let newArr = [...arr, 40];

console.log(arr);    // original
console.log(newArr); // new

Output:
[10, 20, 30]
[10, 20, 30, 40]


🧠 Real Meaning
New memory → safe copy


🔥 MOST IMPORTANT BUG (Beginner Trap)
👉 Reference Problem
let arr1 = [1, 2, 3];
let arr2 = arr1;

arr2.push(4);

console.log(arr1);

Output:
[1, 2, 3, 4] ❗


😱 Why?
👉 Because:
arr1 and arr2 point to SAME memory


✅ Solution (Copy Properly)
let arr1 = [1, 2, 3];
let arr2 = [...arr1];  // copy

arr2.push(4);

console.log(arr1); // safe
console.log(arr2);

3️⃣ Strings are Immutable
👉 Example
let str = "hello";

str[0] = "H";  // ❌ not allowed

console.log(str);

Output:
hello


✅ Correct Way
let newStr = "H" + str.slice(1);
