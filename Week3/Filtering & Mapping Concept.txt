 9. Filtering & Mapping Concept (Intro)
👉 This is one of the MOST USED concepts in modern development (React, APIs, Backend)

🧠 Big Idea
👉 You have data
👉 You want specific part or modified version
That’s where:
Filter → Select data
Map → Transform data


1️⃣ Filtering (Select Specific Data)
👉 Concept
Pick elements that match a condition

Example
let numbers = [10, 20, 30, 40, 50];

let result = numbers.filter(num => num > 25);

console.log(result);

Output:
[30, 40, 50]


🧠 Real-World Use
🛒 Products above ₹1000
🎓 Students who passed
🔍 Search results
📊 Data filtering

2️⃣ Mapping (Transform Data)
👉 Concept
Change each element into something new

Example
let numbers = [1, 2, 3, 4];

let result = numbers.map(num => num * 2);

console.log(result);

Output:
[2, 4, 6, 8]


🧠 Real-World Use
Convert prices (₹ → $)
Format names
Create UI elements
Process API data

🔥 Filter vs Map (Very Important)
Feature
filter()
map()
Purpose
Select data
Transform data
Output size
Same or smaller
Always same
Condition
Yes
No


🔥 Real Example (Array of Objects)
👉 Students Example
let students = [
  {name: "Rahul", marks: 80},
  {name: "Amit", marks: 40},
  {name: "Riya", marks: 90}
];


🎯 Step 1: Filter passed students
let passed = students.filter(s => s.marks > 50);


🎯 Step 2: Get only names
let names = passed.map(s => s.name);

console.log(names);

Output:
["Rahul", "Riya"]


🧠 Powerful Concept (Pipeline Thinking)
let result = students
    .filter(s => s.marks > 50)
    .map(s => s.name);

👉 This is called data pipeline

⚠️ Common Mistakes
❌ Forgetting return
arr.map(num => { num * 2 }) ❌


✅ Correct
arr.map(num => num * 2)


❌ Using map instead of filter

❌ Mutating original data unintentionally
