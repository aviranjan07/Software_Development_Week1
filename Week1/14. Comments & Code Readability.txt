I will explain **Comments & Code Readability in Java** in a **very simple beginner way**.
This topic is important because **good code should be easy to read and understand**.

Good programmers do not only write code that **works**, they write code that **humans can understand**.

---

# 1. Writing Comments

## What is a Comment?

A **comment is text written inside the code that the computer ignores**.

It is used to **explain the code for humans**.

The program **does not execute comments**.

Example:

```java
// This program prints Hello World
System.out.println("Hello World");
```

The line starting with `//` is a **comment**.

---

## Why Comments Are Important

Comments help:

* Explain **what the code does**
* Help other programmers understand the program
* Help you remember the logic later

Example:

```java
// calculate total price
int total = price * quantity;
```

Without the comment, someone might not understand the purpose.

---

## Types of Comments in Java

### 1. Single Line Comment

Used for **short explanations**.

Syntax:

```java
// comment
```

Example:

```java
// print student name
System.out.println(name);
```

---

### 2. Multi-Line Comment

Used for **long explanations**.

Syntax:

```java
/*
 comment
 comment
*/
```

Example:

```java
/*
This program calculates
the area of a circle
*/
```

---

### 3. Documentation Comment

Used to describe **classes and methods**.

Syntax:

```java
/**
 documentation comment
*/
```

Example:

```java
/**
 * This method calculates sum
 */
```

# 2. Clean Code Formatting

Clean formatting makes code **easy to read**.

Bad formatting makes code **confusing**.

---

## Example of Bad Formatting

```java
public class Main{public static void main(String[]args){int a=10;int b=20;System.out.println(a+b);}}
```

Very hard to read.

---

## Example of Clean Formatting

```java
public class Main {

    public static void main(String[] args) {

        int a = 10;
        int b = 20;

        System.out.println(a + b);

    }

}
```

Much easier to understand.

---

## Good Formatting Rules

### 1. Use Proper Indentation

Indent code blocks.

Example:

```java
if(age >= 18)
{
    System.out.println("Adult");
}
```

---

### 2. Use Spaces Around Operators

Bad:

```java
int sum=a+b;
```

Good:

```java
int sum = a + b;
```

---

### 3. Separate Code Blocks

Leave blank lines between sections.

Example:

```java
int a = 10;
int b = 20;

int sum = a + b;

System.out.println(sum);
```

---

# 3. Naming Variables Properly

Variable names should **describe what the variable stores**.

Bad names make code confusing.

---

## Bad Variable Names

```java
int x = 20;
int y = 5;
int z = x * y;
```

What are x, y, z?

Hard to understand.

---

## Good Variable Names

```java
int price = 20;
int quantity = 5;

int totalPrice = price * quantity;
```

Now the code explains itself.

---

## Variable Naming Rules

### 1. Use Meaningful Names

Bad:

```java
int a;
```

Good:

```java
int studentAge;
```

---

### 2. Use Camel Case

Java uses **camelCase**.

Example:

```java
studentAge
totalPrice
userName
```

First word **lowercase**, next words **capitalized**.

---

### 3. Avoid Very Long Names

Bad:

```java
int totalPriceOfAllItemsInsideShoppingCart;
```

Better:

```java
int totalPrice;
```

---

### 4. Avoid Single Letters (Except Loops)

Bad:

```java
int a = 10;
```

Better:

```java
int age = 10;
```

Exception:

```java
for(int i = 0; i < 10; i++)
```

Using `i` in loops is normal.

---

# 4. Example Combining Everything

```java
public class Main {

    public static void main(String[] args) {

        // store product price
        int price = 100;

        // store quantity
        int quantity = 3;

        // calculate total price
        int totalPrice = price * quantity;

        // print result
        System.out.println("Total price = " + totalPrice);

    }

}
```

This program is:

* Easy to read
* Easy to understand
* Easy to maintain

---

# Final Simple Summary

Good readable code follows **three rules**.

### 1. Comments

Explain the code.

```java
// calculate total
```

---

### 2. Clean Formatting

Use indentation and spaces.

Example:

```java
int sum = a + b;
```

---

### 3. Good Variable Names

Use meaningful names.

Example:

```java
int studentAge;
```

---

✅ Simple rule used by professional programmers:

> **Code should be written for humans first, computers second.**
