# 🔹 2. Loop Control Statements — Deep Understanding

Loop control statements **control how loops behave while running**.
Sometimes we need to **stop a loop early**, **skip some iterations**, or **do nothing temporarily**.

These statements help programmers **avoid infinite loops and control program flow**.

Main loop control statements:

```
break
continue
pass (language specific)
```

Used inside loops like:

```
for loop
while loop
do-while loop
```

---

# 1️⃣ break Statement

`break` **immediately stops the loop** and exits it.

When the program encounters `break`, the loop **terminates instantly**, and execution moves to the **next line after the loop**.

### Syntax

```java
break;
```

---

## Example 1 — Simple break

```java
for(int i = 1; i <= 10; i++){

    if(i == 5){
        break;
    }

    System.out.println(i);
}
```

### Output

```
1
2
3
4
```

### Explanation

Loop starts from **1 to 10**

When `i == 5`

```
break executes
loop stops immediately
```

So **5 to 10 never execute**.

---

## Real Example — Password Attempts

```java
int attempts = 1;

while(attempts <= 3){

    System.out.println("Checking password");

    if(attempts == 2){
        System.out.println("Login successful");
        break;
    }

    attempts++;
}
```

### Output

```
Checking password
Checking password
Login successful
```

The loop stops after **successful login**.

---

# 2️⃣ continue Statement

`continue` **skips the current iteration** and moves to the **next loop iteration**.

The loop **does NOT stop**, it just **skips that one step**.

### Syntax

```java
continue;
```

---

## Example — Skip number 5

```java
for(int i = 1; i <= 10; i++){

    if(i == 5){
        continue;
    }

    System.out.println(i);
}
```

### Output

```
1
2
3
4
6
7
8
9
10
```

Explanation:

```
When i = 5
continue runs
System.out.println skipped
Loop continues with i = 6
```

---

## Real Example — Skip Failed Data

Imagine processing student marks.

```java
for(int i = 1; i <= 5; i++){

    if(i == 3){
        continue;
    }

    System.out.println("Processing student " + i);
}
```

### Output

```
Processing student 1
Processing student 2
Processing student 4
Processing student 5
```

Student **3 skipped**.

---

# 3️⃣ pass Statement (Language Specific)

`pass` means **do nothing**.

It is mainly used in **Python**, not Java.

Example in Python:

```python
for i in range(5):
    if i == 3:
        pass
    print(i)
```

Output

```
0
1
2
3
4
```

Explanation:

```
pass → placeholder
program continues normally
```

Developers use `pass` when **logic will be added later**.

Example:

```python
if condition:
    pass
```

This prevents **syntax error when block is empty**.

Java does **not require `pass`**.

---

# 🔹 Difference Between break and continue

| Feature         | break     | continue       |
| --------------- | --------- | -------------- |
| Stops loop      | Yes       | No             |
| Skips iteration | No        | Yes            |
| Loop continues  | No        | Yes            |
| Use case        | Exit loop | Skip condition |

---

# 1️⃣ Nested Loops (Very Important)

A **nested loop** means:

> A loop **inside another loop**.

The **outer loop controls rows**, and the **inner loop controls columns or repeated actions**.

### Structure

```java
for(initialization; condition; update){   // Outer loop

    for(initialization; condition; update){   // Inner loop
        // code
    }

}
```

---

## Visual Understanding

Example:

```java
for(int i=1; i<=3; i++){
    for(int j=1; j<=3; j++){
        System.out.println(i + " " + j);
    }
}
```

### Output

```
1 1
1 2
1 3
2 1
2 2
2 3
3 1
3 2
3 3
```

### Execution Flow

```
Outer Loop → i = 1
    Inner Loop → j = 1,2,3

Outer Loop → i = 2
    Inner Loop → j = 1,2,3

Outer Loop → i = 3
    Inner Loop → j = 1,2,3
```

Meaning:

```
Outer loop runs 3 times
Inner loop runs 3 × 3 = 9 times
```

---

## Real Example — Multiplication Table

```java
for(int i=1; i<=5; i++){

    for(int j=1; j<=10; j++){
        System.out.println(i + " x " + j + " = " + (i*j));
    }

    System.out.println();
}
```

Output:

```
1 x 1 = 1
1 x 2 = 2
...

2 x 1 = 2
...
```

Nested loops are used for:

```
matrix operations
tables
patterns
grid systems
game boards
```

# 2️⃣ Loop Optimization

Loop optimization means:

> **Making loops faster and more efficient**

Developers try to **reduce unnecessary iterations**.

---

## Bad Example (Inefficient)

```java
for(int i=1; i<=1000000; i++){
    if(i == 5){
        System.out.println("Found");
    }
}
```

Even after finding `5`, loop continues **1 million times**.

---

## Optimized Version

```java
for(int i=1; i<=1000000; i++){

    if(i == 5){
        System.out.println("Found");
        break;
    }

}
```

Now loop **stops immediately**.

Much faster.

---

## Another Optimization Example

Bad:

```java
for(int i=0; i<list.size(); i++){
}
```

Better:

```java
int size = list.size();

for(int i=0; i<size; i++){
}
```

Because `list.size()` is not recalculated every loop.

---

## Developer Optimization Techniques

```
Use break to stop early
Reduce nested loops when possible
Avoid unnecessary calculations
Use correct loop conditions
```

# 4️⃣ Infinite Loop Debugging

An **infinite loop** means the loop **never stops**.

Example:

```java
while(true){
    System.out.println("Running...");
}
```

This runs **forever**.

---

## Common Causes

### 1️⃣ Missing Update

Wrong:

```java
int i = 1;

while(i <= 5){
    System.out.println(i);
}
```

`i` never changes → infinite loop.

Correct:

```java
int i = 1;

while(i <= 5){
    System.out.println(i);
    i++;
}
```

---

### 2️⃣ Wrong Condition

Wrong:

```java
for(int i=10; i>0; i++){
}
```

`i++` makes loop infinite.

Correct:

```
i--
```

---

### 3️⃣ Logic Error

```java
while(userInput != password){
}
```

If userInput never changes → infinite loop.

---

## Debugging Tips

Print variable values.

```java
System.out.println("i = " + i);
```

Or use **break temporarily**.
