# 🔹 6. Local vs Global Scope (Deep Clarity)

```text
Local Scope
Global Scope
```

Understanding scope is **very important for debugging and writing clean programs**.

---

# 1️⃣ Local Scope

A **local variable** is declared **inside a function or block**.

It can only be used **inside that function or block**.

### Example

```java
void greet(){

    int age = 25;   // local variable

    System.out.println(age);

}
```

Explanation:

```text
age exists only inside greet()
outside greet() → error
```

---

### Example Showing Error

```java
void greet(){

    int age = 25;

}

public static void main(String[] args){

    System.out.println(age);  // ERROR

}
```

Because:

```text
age belongs only to greet()
```

---

# 2️⃣ Global Scope

A **global variable** is declared **outside functions**.

It can be used **anywhere in the program** (within the class).

### Example

```java
class Example{

    static int number = 100;   // global variable

    public static void main(String[] args){

        System.out.println(number);

    }

}
```

Explanation:

```text
number is accessible everywhere in the class
```

---

# Visual Understanding

```text
Global Variable
   ↓
Available to all functions
```

```
Local Variable
   ↓
Only inside that function
```

---

# 3️⃣ Variable Lifetime

Variable **lifetime** means:

> How long a variable exists in memory.

---

## Local Variable Lifetime

Local variables exist **only during function execution**.

Example:

```java
void test(){

    int x = 10;

}
```

Lifetime:

```text
Function starts → variable created
Function ends → variable destroyed
```

---

## Global Variable Lifetime

Global variables exist **throughout the program execution**.

Example:

```java
static int count = 0;
```

Lifetime:

```text
Program starts → variable created
Program ends → variable destroyed
```

---

# 4️⃣ Shadowing Problem

**Shadowing** happens when a **local variable has the same name as a global variable**.

The **local variable hides the global variable**.

---

### Example

```java
class Example{

    static int number = 50;  // global

    void show(){

        int number = 10;     // local

        System.out.println(number);

    }

}
```

Output:

```text
10
```

Explanation:

```text
local variable hides global variable
```

This is called **variable shadowing**.

---

### Accessing Global Variable

In Java we can use:

```text
ClassName.variable
```

Example:

```java
System.out.println(Example.number);
```

---

# 5️⃣ Why Global Variables Are Risky

Professional developers try to **avoid global variables**.

Because they create **many problems**.

---

## Problem 1 — Hard to Debug

Example:

```java
int balance = 1000;
```

If many functions change this variable:

```text
deposit()
withdraw()
transfer()
interest()
```

Finding where the value changed becomes **very difficult**.

---

## Problem 2 — Unexpected Changes

Example:

```java
balance = 1000
```

One function changes it:

```text
balance = -500
```

Other parts of program may break.

---

## Problem 3 — Security Risk

Global variables can be **modified from anywhere**.

This creates:

```text
bugs
data corruption
security issues
```

---

## Problem 4 — Difficult to Maintain

Large software systems may have:

```text
100+ functions
```

If all access the same global variable:

```text
code becomes chaotic
```

---

# 6️⃣ Developer Best Practice

Professional developers prefer:

```text
local variables
function parameters
return values
```

Example:

### Bad (global)

```java
int total;

void calculate(){
    total = 10 + 5;
}
```

---

### Good (local + return)

```java
int calculate(int a,int b){

    int total = a + b;

    return total;

}
```

Cleaner and safer.

---

# 7️⃣ Real World Example

Imagine a **banking system**.

Bad design:

```text
global balance variable
```

Any function could change it.

Good design:

```text
withdraw(balance, amount)
deposit(balance, amount)
```

Balance is passed **as parameter**.

This prevents **unexpected changes**.

---

# 8️⃣ Common Beginner Mistakes

### 1️⃣ Using global variables everywhere

Beginners often do:

```text
global variables for everything
```

This makes programs **messy**.

---

### 2️⃣ Forgetting variable scope

Example:

```java
for(int i=0;i<5;i++){

}

System.out.println(i);  // ERROR
```

Because:

```text
i only exists inside the loop
```

---

### 3️⃣ Shadowing confusion

```java
int x = 10;

void test(){

    int x = 20;

}
```

Beginners expect **10**, but result is **20**.

---

# 9️⃣ Best Practice Summary

| Rule                   | Explanation                  |
| ---------------------- | ---------------------------- |
| Prefer local variables | safer and easier to debug    |
| Use parameters         | pass data to functions       |
| Avoid global variables | prevents bugs                |
| Use clear names        | prevents shadowing confusion |

---

# 🔟 Developer Mental Model

Professional programmers think like this:

```text
Where should this variable live?
```

Decision:

```text
only needed inside function → local
needed across program → carefully global
```

| Local scope  | variable inside function             |
| Global scope | variable accessible everywhere       |
| Lifetime     | how long variable exists             |
| Shadowing    | local variable hides global variable |
| Global risk  | debugging + security issues          |

```text
Use global variables only when absolutely necessary
```

Because **good software architecture depends on controlled data flow**.
