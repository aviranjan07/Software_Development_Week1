## 1️⃣ Comparison & Logical Operators — Deep Understanding (Beginner → Real Programming)

These operators are **the foundation of decision making in programming**.
They help a program **compare values and decide what action to take**.

Example real-life decisions:

* ATM checks **balance >= withdrawal amount**
* Login system checks **username AND password**
* Form validation checks **email OR phone number provided**

Without these operators, programs **cannot make intelligent decisions**.

---

# Part 1 — Comparison Operators

Comparison operators **compare two values** and return **true or false**.

| Operator | Meaning          | Example    | Result |
| -------- | ---------------- | ---------- | ------ |
| `==`     | Equal to         | `5 == 5`   | true   |
| `!=`     | Not equal        | `5 != 3`   | true   |
| `>`      | Greater than     | `10 > 5`   | true   |
| `<`      | Less than        | `2 < 8`    | true   |
| `>=`     | Greater or equal | `10 >= 10` | true   |
| `<=`     | Less or equal    | `5 <= 10`  | true   |

### Example (Java)

```java
int a = 10;
int b = 20;

System.out.println(a == b); 
System.out.println(a < b);
System.out.println(a > b);
```

Output

```
false
true
false
```

Explanation

```
10 == 20 → false
10 < 20 → true
10 > 20 → false
```

# Important Concept

## `==` vs `=` (Very Common Beginner Mistake)

| Symbol | Meaning    |
| ------ | ---------- |
| `=`    | Assignment |
| `==`   | Comparison |

### Example

Wrong:

```java
if(a = 5)
```

Correct:

```java
if(a == 5)
```

Explanation:

```
a = 5  → assign value
a == 5 → compare value
```

Many beginners **confuse this and get errors**.

---

# Part 2 — `==` vs `===` (Important Concept)

Some languages support **strict comparison (`===`)**.

| Operator | Meaning                   |
| -------- | ------------------------- |
| `==`     | Compare values            |
| `===`    | Compare value + data type |

Example (JavaScript concept)

```
5 == "5"   → true
5 === "5"  → false
```

Explanation:

```
== converts type
=== checks type strictly
```

⚠️ Java **does not use `===`**, but understanding this helps when learning **JavaScript later**.

# Part 3 — Logical Operators

Logical operators **combine multiple conditions**.

| Operator | Meaning |   |    |
| -------- | ------- | - | -- |
| `&&`     | AND     |   |    |
| `        |         | ` | OR |
| `!`      | NOT     |   |    |

They return **true or false**.

---

# AND Operator (`&&`)

Condition is **true only if BOTH conditions are true**.

| A     | B     | A && B |
| ----- | ----- | ------ |
| true  | true  | true   |
| true  | false | false  |
| false | true  | false  |
| false | false | false  |

### Example

```java
int age = 25;

if(age > 18 && age < 60){
    System.out.println("Eligible for job");
}
```

Explanation

```
age > 18 → true
age < 60 → true
true && true → true
```

Output

```
Eligible for job
```

---

# OR Operator (`||`)

Condition is **true if at least one condition is true**.

| A | B | A || B |
|---|---|---|
| true | true | true |
| true | false | true |
| false | true | true |
| false | false | false |

### Example

```java
int marks = 35;

if(marks >= 35 || marks == 33){
    System.out.println("Pass");
}
```

Explanation

```
marks >= 35 → true
true || anything → true
```

Output

```
Pass
```

---

# NOT Operator (`!`)

NOT **reverses the result**.

| Condition | Result |
| --------- | ------ |
| true      | false  |
| false     | true   |

### Example

```java
boolean isLoggedIn = false;

if(!isLoggedIn){
    System.out.println("Please login first");
}
```

Explanation

```
!false → true
```

Output

```
Please login first
```

# Part 4 — Combining Multiple Conditions

This is **real programming logic**.

Example:

```java
int age = 25;
boolean hasLicense = true;

if(age >= 18 && hasLicense){
    System.out.println("You can drive");
}
else{
    System.out.println("Not allowed to drive");
}
```

Logic

```
Age >= 18
AND
License available
```

Both must be **true**.

# Operator Priority (Very Important)

Order of execution:

```
1️⃣ !
2️⃣ &&
3️⃣ ||
```

Example

```java
true || false && false
```

Execution

```
false && false → false
true || false → true
```

Result

```
true
```
