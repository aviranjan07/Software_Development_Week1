# 🧠 1. What is an Error Stack (Simple Meaning)

👉 Error stack (stack trace) = **a roadmap of how your code crashed**

It tells you:

* ❌ What error happened
* 📍 Where it happened
* 🔁 How the code reached there

---

# 📌 2. Example Error Stack

```js
function login() {
  getUser();
}

function getUser() {
  let user = undefined;
  console.log(user.name);
}

login();
```

---

### ❌ Error Output:

```
TypeError: Cannot read properties of undefined (reading 'name')
    at getUser (app.js:6)
    at login (app.js:2)
    at app.js:10
```

---

# 🔍 3. How to Read It (MOST IMPORTANT)

👉 Always read **BOTTOM → TOP**

---

### Step-by-step:

#### 🧱 Bottom:

```
at app.js:10
```

👉 Program started here

---

#### ⬆️ Next:

```
at login (app.js:2)
```

👉 `login()` was called

---

#### ⬆️ Top (MOST IMPORTANT):

```
at getUser (app.js:6)
```

👉 💥 ERROR happened here

---

# 🎯 Rule:

> 🔥 The TOP line = where the error actually occurred

---

# 🧠 4. Understanding Error Types

---

## ❌ TypeError

```
Cannot read properties of undefined
```

👉 Accessing something that doesn’t exist

---

## ❌ ReferenceError

```
x is not defined
```

👉 Variable not declared

---

## ❌ SyntaxError

```
Unexpected token
```

👉 Code structure wrong

---

## ❌ RangeError

```
Maximum call stack size exceeded
```

👉 Infinite recursion

---

# 🔁 5. Call Stack (Deep Concept)

👉 Stack = **function call history**

---

### Example Flow:

```
main()
 └── login()
      └── getUser() ❌ error
```

---

👉 Stack trace shows this chain

---

# 🧠 6. Real Example (Frontend)

```js
button.addEventListener("click", handleClick);

function handleClick() {
  processData();
}

function processData() {
  let data = null;
  console.log(data.id);
}
```

---

### Stack:

```
TypeError: Cannot read properties of null (reading 'id')
    at processData (app.js:10)
    at handleClick (app.js:6)
```

---

👉 Means:

* User clicked button
* `handleClick()` ran
* `processData()` crashed

# 🧠 8. Mental Model (VERY IMPORTANT)

When you see stack:

Ask:

1. Where did it break? (top line)
2. Why did it break? (error message)
3. How did it reach there? (rest of stack)

---

# ⚠️ 9. Common Mistakes

❌ Reading top → bottom (wrong)
❌ Ignoring file + line number
❌ Fixing random lines
❌ Not understanding error type

---

# 🧪 10. Practice Exercise

### Bug:

```js
function calculateTotal(price) {
  return price.toFixed(2);
}

console.log(calculateTotal(undefined));
```

---

### Error:

```
TypeError: Cannot read properties of undefined (reading 'toFixed')
    at calculateTotal (app.js:2)
```

---

👉 Fix:

```js
if (price !== undefined) {
  return price.toFixed(2);
}
```

---

# 💼 11. Real Developer Trick

👉 When stack is long:

* Ignore library/framework lines
* Focus on YOUR file (`app.js`, `index.js`)

---

# 🔥 12. Pro Tip
