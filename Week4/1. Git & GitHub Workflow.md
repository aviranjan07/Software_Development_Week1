🚀 1. What is Git (Simple Explanation)
👉 Git is like a time machine for your code
You can save versions
Go back anytime
Track changes
Work safely without losing code
📦 Example:
You wrote code today → broke it tomorrow → Git lets you go back

🌐 2. What is GitHub
👉 GitHub is like Google Drive for code + collaboration
Stores your Git projects online
Helps teams work together
Shows your portfolio (VERY important for jobs)

🔁 3. Complete Git Workflow (Most Important)
🧠 Think like this:
Working Directory → Staging Area → Repository → GitHub


📍 Step-by-step flow:
1. Initialize Git
git init

👉 Start tracking your project

2. Check Status
git status

👉 See what changed

3. Add Files (Staging Area)
git add .

👉 Prepare files for commit

4. Commit (Save Version)
git commit -m "Added login feature"

👉 Save snapshot

5. Connect to GitHub
git remote add origin <repo-link>


6. Push Code to GitHub
git push -u origin main


🔁 After first push (daily workflow)
git add .
git commit -m "your message"
git push


🌿 4. Branching (Real Developer Skill)
👉 Branch = separate workspace
Create branch:
git branch feature-login

Switch branch:
git checkout feature-login

👉 OR (modern way):
git checkout -b feature-login


Merge branch:
git checkout main
git merge feature-login


🔄 5. Pull (Get latest code)
git pull origin main

👉 Always pull before starting work in team projects

⚠️ 6. Common Mistakes (VERY IMPORTANT)
❌ Forgetting git add
❌ Bad commit messages → "update", "fix" (avoid this)
❌ Pushing directly to main branch
❌ Not pulling before working
❌ Panic during conflicts

💥 7. Merge Conflict (Real Life Situation)
👉 Happens when:
Two people change same file
Fix:
Open file
Choose correct code
Remove conflict markers
Then:
git add .
git commit -m "resolved conflict"


🧠 8. Professional Commit Messages
❌ Bad:
update code

✅ Good:

Fixed navbar responsive issue
Improved API error handling
