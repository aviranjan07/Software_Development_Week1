# 🏗️ Project Structure (Standard)

```text
main        → 🚀 Production (live code)
develop     → 🧪 Integration (optional in small teams)
feature/*   → ✨ New features
bugfix/*    → 🐛 Fix issues
hotfix/*    → 🚨 Urgent fixes
```

👉 Small teams often use only:

```text
main + feature branches
```

---

# 🔄 FULL REAL WORKFLOW (Step-by-Step)

---

## 🟢 1. Start your day (ALWAYS)

```bash
git switch main
git pull origin main
```

👉 Get latest code from team

---

## 🌱 2. Create your branch

```bash
git switch -c feature/login
```

👉 Never work on `main`

---

## ✏️ 3. Work + Commit (small commits)

```bash
git add .
git commit -m "Add login form UI"
```

👉 Good commit = clear message

---

## 🔄 4. Sync with main (VERY IMPORTANT)

```bash
git fetch origin
git rebase origin/main
```

👉 Avoid future conflicts

---

## 📤 5. Push your branch

```bash
git push origin feature/login
```

---

## 🔁 6. Create Pull Request (PR)

On GitHub:

* Compare `feature/login` → `main`
* Add description
* Request review

👉 This is where teams collaborate 👀

---

## 🔍 7. Code Review

* Teammates check your code
* Suggest improvements
* You update commits

---

## 🔀 8. Merge

After approval:

👉 Merge PR into `main`

---

## 🧹 9. Cleanup

```bash
git branch -d feature/login
git push origin --delete feature/login
```

# ⚠️ Real-World Rules (VERY IMPORTANT)

---

## ❌ NEVER do this

```bash
git push origin main
```

👉 Direct push to main = ❌ (in teams)

---

## ✅ ALWAYS do this

✔ Use feature branches
✔ Use Pull Requests
✔ Get code review

---

# 🔁 Handling Conflicts (Real Scenario)

When rebase/merge fails:

```bash
git status
```

👉 Fix files manually → then:

```bash
git add .
git rebase --continue
```

---

# 🧩 When to use what?

| Situation             | Command               |
| --------------------- | --------------------- |
| Check changes         | `git diff`            |
| See history           | `git log`             |
| Save work temporarily | `git stash`           |
| Undo mistake          | `git restore / reset` |
| Update code           | `git fetch / pull`    |

# 🏆 Pro Developer Tips

---

## 1️⃣ Small commits > big commits

👉 Easier review & debugging

---

## 2️⃣ Meaningful messages

```bash
"Fix login validation bug"
```

❌ Not:

```bash
"update"
```

---

## 3️⃣ Pull/Rebase frequently

👉 Avoid huge conflicts

---

## 4️⃣ Use `.gitignore`

👉 Keep repo clean

---

## 5️⃣ Use `--rebase` for clean history

```bash
git pull --rebase
```
