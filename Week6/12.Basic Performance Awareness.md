## 🧠 1. Core Idea

👉 Browsers are **fast… but DOM operations are expensive**

So rule:

> “Minimize DOM work, maximize efficiency”

---

# 🔹 2. Avoid Unnecessary DOM Updates

## ❌ Bad (Slow Code)

```js id="v8t2k4"
for (let i = 0; i < 100; i++) {
  const li = document.createElement("li");
  li.textContent = i;
  document.querySelector("ul").appendChild(li);
}
```

👉 Problem:

* DOM accessed 100 times ❌
* Causes reflow/repaint repeatedly ❌

---

## ✅ Good (Optimized)

```js id="n4k7x1"
const ul = document.querySelector("ul");

for (let i = 0; i < 100; i++) {
  const li = document.createElement("li");
  li.textContent = i;
  ul.appendChild(li);
}
```

👉 DOM accessed once ✅

---

## 🔥 Even Better (Batch Updates)

```js id="w2m9q6"
const fragment = document.createDocumentFragment();

for (let i = 0; i < 100; i++) {
  const li = document.createElement("li");
  li.textContent = i;
  fragment.appendChild(li);
}

ul.appendChild(fragment);
```

👉 One DOM update = best performance 🚀

# 🔹 3. Cache Selectors (VERY IMPORTANT)

## ❌ Bad

```js id="f7k3m2"
document.querySelector("#title").textContent = "Hi";
document.querySelector("#title").style.color = "red";
```

👉 Querying DOM multiple times ❌

---

## ✅ Good

```js id="z9p4x8"
const title = document.querySelector("#title");

title.textContent = "Hi";
title.style.color = "red";
```

👉 Cached = faster + cleaner ✅

---

# 🔹 4. Avoid Layout Thrashing

👉 Happens when:

* You read + write DOM repeatedly

---

## ❌ Bad

```js id="m5x8q2"
element.style.width = "100px";
console.log(element.offsetWidth);
element.style.width = "200px";
```

---

## ✅ Better

```js id="c3v9k1"
const width = element.offsetWidth;

element.style.width = "200px";
```

---

# 🔥 5. Use classList Instead of style

## ❌ Bad

```js id="q8n2w7"
element.style.color = "red";
element.style.backgroundColor = "black";
```

---

## ✅ Good

```js id="r6k1p9"
element.classList.add("dark-mode");
```

👉 Cleaner + better performance
