## 🧠 1. Core Idea (Very Important)

When you click something, the event doesn’t just stay there ❗

👉 It **travels through the DOM**

This journey = **Event Flow (Propagation)**

---

### 📌 3 Phases of Event Flow

1. **Capturing Phase** (top → down)
2. **Target Phase** (actual element)
3. **Bubbling Phase** (bottom → up)

---

## 🔹 2. Event Bubbling (Default Behavior 🔥)

👉 Flow:
**Child → Parent → Document**

---

### 💻 Example

```html id="2x6hgt"
<div id="parent">
  <button id="child">Click Me</button>
</div>
```

```js id="pq5u9x"
document.getElementById("parent")
  .addEventListener("click", () => console.log("PARENT"));

document.getElementById("child")
  .addEventListener("click", () => console.log("CHILD"));
```

👉 Click button → Output:

```
CHILD
PARENT
```

---

### 💡 Why It Happens?

Because events **bubble up the DOM tree**

---

## 🔹 3. Event Capturing (Intro)

👉 Flow:
**Parent → Child**

---

### 💻 Enable Capturing

```js id="ks0q3r"
parent.addEventListener("click", () => {
  console.log("PARENT (capturing)");
}, true);
```

👉 `true` = capturing phase

---

### ⚠️ Important

* Rarely used in real apps
* But asked in interviews

---

## 🔥 4. Event Delegation (GAME CHANGER)

👉 Instead of adding many listeners → add **one listener to parent**

---

### 💻 Example

```js id="93oysu"
ul.addEventListener("click", function(e) {
  if (e.target.tagName === "LI") {
    console.log("Clicked:", e.target.textContent);
  }
});
```

---

### 💡 Why This is Powerful?

Without delegation ❌:

* 100 items → 100 listeners

With delegation ✅:

* 100 items → 1 listener

---

### 🔥 Real Use Cases

* Todo list
* Dynamic product list
* Chat messages
* Comments section

---

## ⚠️ 5. Common Mistakes

❌ Not understanding bubbling → duplicate logs
❌ Adding listeners to each element (bad performance)
❌ Forgetting `event.target` in delegation
❌ Confusing capturing vs bubbling

---

## 🧩 6. Interview / Pro Insights

You MUST know:

### 🔹 Bubbling vs Capturing

| Feature   | Bubbling       | Capturing      |
| --------- | -------------- | -------------- |
| Direction | Child → Parent | Parent → Child |
| Default   | ✅ Yes          | ❌ No           |

---

### 🔹 stopPropagation()

```js id="sydp6t"
event.stopPropagation();
```

👉 Stops event from moving further

---

### 🔥 Advanced Insight

Libraries like React:

* Use event delegation internally
* Attach events at root for performance

---

### 💡 Pro Line

> “I use event delegation to handle dynamic elements efficiently and avoid unnecessary listeners.”
