## 🧠 1. Core Concept (Simple + Deep)

**Local Storage** = a way to store data **inside the user’s browser**.

👉 Key points:

* Stores **key-value pairs**
* Data stays even after refresh or closing browser
* Works only on client-side (frontend)

Think:

> localStorage = mini database inside browser

---

## ⚙️ 2. Real-World Use

Used in almost every app:

* Save login state (remember me)
* Store cart items (Amazon 🛒)
* Dark/light mode preference
* Save form drafts
* Offline support

---

## 💻 3. Core Methods (Must Know)

### 🔹 Store Data

```js id="n9k1y2"
localStorage.setItem("name", "Avi");
```

---

### 🔹 Get Data

```js id="k3m8xq"
const name = localStorage.getItem("name");
console.log(name);
```

---

### 🔹 Remove Data

```js id="z7v2lp"
localStorage.removeItem("name");
```

---

### 🔹 Clear All

```js id="o4t8rs"
localStorage.clear();
```

---

## 🔥 4. IMPORTANT: Store Objects (JSON)

LocalStorage only stores **strings** ❗

👉 So we use JSON:

### Store Object

```js id="s1x9bz"
const user = { name: "Avi", age: 22 };

localStorage.setItem("user", JSON.stringify(user));
```

---

### Get Object

```js id="w6c3pd"
const data = JSON.parse(localStorage.getItem("user"));
console.log(data.name);
```

---

## ⚠️ 5. Common Mistakes (VERY IMPORTANT)

❌ Forgetting `JSON.stringify()`
❌ Forgetting `JSON.parse()`
❌ Storing sensitive data (passwords ❌)
❌ Assuming unlimited storage (~5MB limit)

👉 Golden Rule:

> Never store passwords or tokens in localStorage

## 🧩 6. Interview / FAANG Insights

You should know:

### 🔹 localStorage vs sessionStorage

| Feature | localStorage | sessionStorage |
| ------- | ------------ | -------------- |
| Expiry  | Never        | Tab close      |
| Scope   | All tabs     | One tab        |

---

### 🔹 Cookies vs LocalStorage

* Cookies → sent to server
* LocalStorage → only browser

---

### 🔹 Security Concept

LocalStorage is vulnerable to:

* **XSS (Cross-Site Scripting)**
