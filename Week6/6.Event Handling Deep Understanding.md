## 🧠 1. Core Idea

Event handling =
👉 “How your app reacts when the user does something”

Without this:

* User clicks → nothing happens ❌
  With this:
* User clicks → action triggered ✅

---

# 🔹 2. Important Events You MUST Know

---

## ✅ click (Most Basic)

```js
button.addEventListener("click", () => {
  console.log("Clicked 🚀");
});
```

👉 Used for:

* Buttons
* Delete / Add actions
* UI toggles

---

## ✅ input (Real-time typing)

```js
input.addEventListener("input", (e) => {
  console.log(e.target.value);
});
```

👉 Fires on:

* Every keystroke

👉 Used for:

* Live search 🔍
* Form validation

---

## ✅ change (After user finishes)

```js
input.addEventListener("change", (e) => {
  console.log(e.target.value);
});
```

👉 Fires when:

* Input loses focus (blur)

👉 Used for:

* Dropdowns
* Final value confirmation

---

## ✅ submit (Forms 🔥 VERY IMPORTANT)

```js
form.addEventListener("submit", (e) => {
  e.preventDefault();
  console.log("Form submitted");
});
```

👉 Default behavior:

* Page reload ❌

👉 We stop it using:

```js
e.preventDefault();
```

---

# 🔥 3. Event Object (Super Important)

Every event gives an **event object**

```js
button.addEventListener("click", (event) => {
  console.log(event);
});
```

---

## 🔑 Most Useful Properties

```js
event.target        // actual clicked element
event.type          // event name
event.value         // input value (via target)
```

---

### 🔥 Difference (Interview Favorite)

| Property      | Meaning                   |
| ------------- | ------------------------- |
| target        | actual element clicked    |
| currentTarget | element listener attached |

---

# 🔹 4. preventDefault() (CRITICAL)

👉 Stops browser default behavior

---

### Example: Form

```js
form.addEventListener("submit", (e) => {
  e.preventDefault(); // stops reload

  console.log("Now I control the form 😎");
});
```

---

### Example: Link

```js
link.addEventListener("click", (e) => {
  e.preventDefault(); // stops navigation
});
```

---

# ⚠️ 5. Common Mistakes

❌ Forgetting `preventDefault()` → page reload
❌ Using `change` instead of `input` for live updates
❌ Not using `event.target` properly
❌ Adding listeners inside loops unnecessarily
