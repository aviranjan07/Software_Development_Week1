## 🧠 1. Core Concept (Deep Understanding)

Before code, understand this:

👉 DOM = **Tree Structure**

```
html
 └── body
      ├── div
      │    ├── h1
      │    └── p
      └── ul
           ├── li
           ├── li
           └── li
```

So selection = **finding the right node in this tree**

---

## 🔹 2. getElementById (Fastest + Simple)

```js
const title = document.getElementById("title");
```

### ✅ When to use:

* Unique element
* High performance needed

### ⚠️ Limitation:

* Only works with `id`

👉 Think:

> “Direct shortcut to element”

---

## 🔹 3. querySelector (Most Powerful 🔥)

```js
const item = document.querySelector(".item");
```

👉 Uses CSS selectors:

```js
document.querySelector("#id")
document.querySelector(".class")
document.querySelector("div p")
```

### ✅ Why developers love it:

* Flexible
* Cleaner
* Works like CSS

---

## 🔹 4. querySelectorAll (Multiple Elements)

```js
const items = document.querySelectorAll(".item");
```

👉 Returns: **NodeList**

```js
items.forEach(el => console.log(el));
```

---

## ⚠️ Important Difference

| Method           | Returns             |
| ---------------- | ------------------- |
| getElementById   | Single element      |
| querySelector    | First match         |
| querySelectorAll | NodeList (multiple) |

## 🔥 5. DOM Traversing (REAL POWER)

This is where beginners struggle — you won’t.

---

### 🔹 Parent

```js
element.parentElement
```

---

### 🔹 Children

```js
element.children
```

---

### 🔹 First / Last Child

```js
element.firstElementChild
element.lastElementChild
```

---

### 🔹 Siblings

```js
element.nextElementSibling
element.previousElementSibling
```

---

## 💻 Real Example

```html
<ul id="list">
  <li>Item 1</li>
  <li class="active">Item 2</li>
  <li>Item 3</li>
</ul>
```

```js
const active = document.querySelector(".active");

console.log(active.parentElement);          // ul
console.log(active.nextElementSibling);     // Item 3
console.log(active.previousElementSibling); // Item 1
```

## ⚠️ 6. Common Mistakes (Kills Beginners)

❌ Using wrong selector (`.` vs `#`)
❌ Forgetting `querySelectorAll` returns NodeList
❌ Not checking null (element not found)
❌ Confusing `children` vs `childNodes`

👉 Pro Tip:

```js
if (element) {
  // safe code
}
```
