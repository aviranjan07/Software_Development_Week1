## 🧠 2. Media Queries (CORE TOOL)

👉 Allows your design to **adapt based on screen size**

---

### 🔹 Basic Syntax

```css
@media (max-width: 768px) {
  body {
    background: lightblue;
  }
}
```

---

### 🔥 Types of Media Queries

#### ✅ Mobile-first (Recommended)

```css
/* Default = mobile */
.container {
  flex-direction: column;
}

/* Tablet and up */
@media (min-width: 768px) {
  .container {
    flex-direction: row;
  }
}
```

---

#### ❌ Desktop-first (less preferred)

```css
@media (max-width: 768px) {
  /* override styles */
}
```

---

### 📏 Common Breakpoints

```css
/* Small phones */
@media (max-width: 480px)

/* Tablets */
@media (min-width: 768px)

/* Laptops */
@media (min-width: 1024px)
```

---

## 🧠 3. Mobile-First Thinking (PRO LEVEL)

> Build for **small screens first**, then expand

---

### 🔥 Why?

* Simpler design
* Better performance
* Easier scaling
* Industry standard

---

### 🧱 Example

```css
/* Mobile */
.card {
  width: 100%;
}

/* Desktop */
@media (min-width: 768px) {
  .card {
    width: 300px;
  }
}
```

---

## 🧠 4. Flexible Units (VERY IMPORTANT)

---

### ❌ Avoid rigid layout

```css
width: 500px;
```

---

### ✅ Use flexible units

```css
width: 100%;
max-width: 500px;
```

---

### 🔹 Key Units

| Unit | Meaning         | Use                  |
| ---- | --------------- | -------------------- |
| %    | Parent-based    | Layout width         |
| rem  | Root-based      | Fonts                |
| em   | Relative        | Components           |
| vw   | Viewport width  | Full-screen sections |
| vh   | Viewport height | Hero sections        |

---

## 🧱 5. Responsive Layout with Flexbox

```css
.container {
  display: flex;
  flex-wrap: wrap;
  gap: 20px;
}

.card {
  flex: 1 1 250px;
}
```

👉 Automatically adjusts layout 🔥

---

## 🧠 6. Responsive Images

```css
img {
  max-width: 100%;
  height: auto;
}
```

👉 Prevents overflow on small screens

---

## ⚠️ 7. Common Mistakes

❌ Not using viewport meta tag
❌ Too many breakpoints (overengineering)
❌ Fixed widths everywhere
❌ Ignoring mobile testing
❌ Using px for fonts

## 💎 10. Pro Developer Tips

✔ Always start mobile-first
✔ Use fewer, meaningful breakpoints
✔ Combine Flexbox + responsive units
✔ Test on real devices
✔ Think: *“Will this break on smaller screens?”*
